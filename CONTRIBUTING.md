# Contributing to Designwear

Thank you for your interest in contributing to Designwear! We welcome contributions from the community to make this project better.

## How to Contribute

### Reporting Bugs
If you find a bug, please create a GitHub Issue including:
-   A clear title and description.
-   Steps to reproduce the issue.
-   Expected vs. actual behavior.
-   Screenshots or logs if applicable.

### Suggesting Enhancements
We love new ideas! Please open an issue to discuss your feature suggestion before implementing it to ensure it aligns with the project goals.

### Pull Requests
1.  **Fork the repository**.
2.  Create a new branch for your feature or fix: `git checkout -b feature/amazing-feature`.
3.  **Commit your changes** with descriptive commit messages.
4.  **Push to your branch**: `git push origin feature/amazing-feature`.
5.  Open a **Pull Request** targeting the `main` branch.

## Code Style
-   We follow standard Flutter/Dart style guidelines.
-   Please run `flutter analyze` and `flutter test` before submitting your PR to ensure the CI pipeline passes.
-   Keep code clean and commented where necessary.
